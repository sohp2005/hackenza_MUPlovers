## 1. Problem Understanding

The task is deceptively simple to state but genuinely hard to solve: given a ~2 minute audio recording, determine whether the speaker is a native or non-native speaker of the language — using only the raw audio signal, no transcription, no manual labeling.

What makes this hard is that native vs non-native differences are not localized to a single moment. They unfold across the entire recording as a combination of:

- **Phonetic patterns** — how individual sounds are produced, which phonemes get substituted or merged
- **Prosody** — the rhythm, pitch contour, and intonation patterns of speech
- **Temporal consistency** — how these patterns repeat and vary across phrases over a long span
- **Pause and hesitation behavior** — where speakers pause, for how long, and how often

A single short clip might be ambiguous. A 2-minute recording gives enough signal to reliably distinguish consistent native-like patterns from non-native ones. Our entire pipeline is designed around exploiting this long-range structure.

---

## 2. Core Insight and Design Philosophy

Most naive approaches treat this as a short-clip classification problem — extract features from the whole recording and classify. We rejected this early.

Our key insight: **accent is a sequence-level phenomenon, not a frame-level one.**

This led us to a chunk-based temporal modeling approach:
- Split each 2-minute recording into overlapping 3-second chunks
- Extract rich acoustic features per chunk
- Model the sequence of chunks with a temporal model (GRU)
- Aggregate using attention that learns which chunks are most informative

This design means the model can learn that a speaker consistently mispronounces certain sounds across phrases, or that their pitch reset pattern after each sentence is non-native-like — patterns that only emerge over time.

## 3. Data

### 3.1 Primary Dataset
The provided training dataset contains labeled audio recordings (~160 files), each approximately 2 minutes long, labeled as native (1) or non-native (0). We performed an 80/20 stratified train/val split, ensuring label distribution is preserved in both splits.

**Label distribution:** The dataset has a class imbalance — more native than non-native samples. We handle this explicitly in training via `pos_weight` in the loss function.

### 3.2 External Datasets
To improve generalization and robustness, we sourced and incorporated additional labeled audio data beyond the provided training set.

**External dataset (explored):** We collected and preprocessed an initial external dataset, running it through our full preprocessing pipeline including chunking, VAD, and prosody extraction. This gave us additional coverage of diverse recording conditions and accents.

**External2 (incorporated into training):** A second external dataset was fully processed through our entire pipeline — preprocessing, chunking, WavLM-Large embedding extraction, prosody, noise features, assembly, and normalization — and combined with the primary training data. This dataset contains 125 unique speakers, adding meaningful diversity to the training distribution.

All external data went through the identical preprocessing pipeline as the primary dataset to ensure consistency.

### 3.3 Test Set
The unlabeled test set provided by the organizers was processed through the same feature extraction pipeline (without labels). Predictions are generated by running inference with the trained model and outputting `predictions.csv`.

## 4. Preprocessing Pipeline

Every audio file — regardless of source — goes through the same automated preprocessing steps:

**Step 1 — Standardization**
- Convert to mono
- Resample to 16 kHz
- Peak amplitude normalization (scaled to 0.95 max)

**Step 2 — Chunking**
- 3-second sliding windows with 1.5-second hop (50% overlap)
- Yields approximately 70–80 chunks per 2-minute file
- Short trailing chunks are zero-padded to 3 seconds

**Step 3 — VAD (Voice Activity Detection)**
- Energy-based frame-level VAD (30ms frames, threshold 0.01 RMS)
- Computes per-chunk: `speech_ratio`, `pause_ratio`, `max_pause_length`
- Used as input features and as weights for downstream processing

This pipeline is fully automated — no manual intervention at any stage.

## 5. Feature Engineering

Each chunk is represented by a concatenation of three feature groups, yielding a **1039-dimensional** feature vector per chunk.

### 5.1 WavLM-Large Embeddings [1024-dim]

We use `microsoft/wavlm-large` — a self-supervised speech model trained on 94,000 hours of audio. Rather than using only the last hidden layer (which captures mostly semantic information), we use a **weighted combination of all 25 hidden layers**:

```
embedding = Σ softmax(w_i) * hidden_state_i   for i in 0..24
```

The layer weights are learnable parameters, allowing the model to discover which layers are most informative for accent classification. Early layers capture low-level acoustic/phonetic features; middle layers capture phoneme-level patterns; later layers capture higher-level structure. Accent information is distributed across all of these.

This is a significant improvement over using only `last_hidden_state`, which is what most baseline approaches do.

### 5.2 Prosody Features [10-dim]

Explicit prosodic features capture rhythm and intonation patterns that embeddings alone may not surface clearly:

| Feature | What it captures |
|---|---|
| F0 mean | Average pitch level |
| F0 std | Pitch variability |
| F0 slope | Pitch trend over chunk |
| F0 range | Pitch dynamic range (P95 - P5) |
| Energy mean | Average loudness |
| Energy std | Loudness variability |
| Speech ratio | Fraction of chunk that is voiced |
| Speaking rate proxy | Onset peaks per second |
| Rhythm variability | Variability of voiced/unvoiced segment lengths |
| Spectral flux | Rate of spectral change (articulation speed) |

Pitch extraction uses `librosa.pyin` (probabilistic YIN algorithm), which is robust to noise. Only voiced frames (high confidence) are used for F0 statistics.

### 5.3 Noise / Recording Quality Features [5-dim]

These features improve robustness across different recording conditions:

| Feature | What it captures |
|---|---|
| SNR proxy | Signal-to-noise ratio estimate |
| Spectral entropy | Randomness of the frequency distribution |
| Spectral flatness | How noise-like vs tonal the signal is |
| Spectral centroid | Center of mass of the spectrum |
| Spectral bandwidth | Spread of energy across frequencies |

Recording quality varies significantly across datasets and speakers. Without these features, the model could confuse recording artifacts with accent-related patterns.

### 5.4 Normalization

All features are normalized using `StandardScaler` fitted on the training set (all chunks stacked). The scaler is saved to `cache/scaler_1039.pkl` and applied consistently to validation and test data.

## 6. Model Architecture

### 6.1 Evolution of Architecture

We iterated through several model architectures, running ablations at each stage:

**Attempt 1 — Mean Pooling Baseline**
Simple mean pooling of WavLM-Base embeddings across all chunks, fed into a linear classifier. Fast and simple but throws away all temporal information.

**Attempt 2 — GRU Only**
Single-layer GRU over chunk embeddings, using the final hidden state. Captures temporal order but the final hidden state is biased toward the end of the recording.

**Attempt 3 — GRU + Simple Attention**
GRU with attention pooling over all hidden states. Attention allows the model to focus on the most accent-informative chunks rather than just the last one. Significant improvement.

**Attempt 4 — Full Pipeline (WavLM-Base, 783-dim)**
Added prosody and noise features to the embedding. The model now sees 768 + 10 + 5 = 783-dim feature vectors per chunk.

**Final Model — Upgraded Architecture (WavLM-Large, 1039-dim)**
Switched to WavLM-Large (1024-dim) with weighted layer combination. Upgraded model architecture with projection layer, 2-layer BiGRU, multi-head attention, and batch normalization.

### 6.2 Final Architecture

```
Input [T, 1039]
    │
    ▼
Linear(1039 → 256) + LayerNorm + ReLU        ← Projection
    │
    ▼
BiGRU(256, hidden=256, layers=2, dropout=0.3) ← Temporal modeling
    │                                            Output: [T, 512]
    ▼
MultiheadAttention(embed=512, heads=4)        ← Accent pattern focus
    │
    ▼
Mean Pool over T → [512]                      ← Aggregation
    │
    ▼
BatchNorm1d(512)                              ← Stability
    │
    ▼
Linear(512→256) → ReLU → Dropout(0.3)        ← Classifier
    │
    ▼
Linear(256→1) → Sigmoid                      ← Output: P(native)
```

**Why each component:**

- **Projection layer** — reduces input dimensionality and creates a shared learned representation space before temporal modeling
- **2-layer BiGRU** — bidirectional so it sees context from both past and future chunks; 2 layers for deeper temporal abstraction
- **Multi-head attention** — 4 heads independently learn to focus on different types of accent-relevant segments
- **BatchNorm** — stabilizes training, especially important with variable-length sequences after pooling
- **Dropout** — regularization against overfitting given limited training data

### 6.3 Training Configuration

| Parameter | Value |
|---|---|
| Loss | `BCEWithLogitsLoss` with `pos_weight` |
| pos_weight | `num_non_native / num_native` (handles class imbalance) |
| Optimizer | Adam, lr=1e-4 |
| Scheduler | ReduceLROnPlateau (patience=3, factor=0.5) |
| Epochs | 50 |
| Batch size | 8 |
| Sequence padding | `pad_sequence` with attention mask |

## 8. Inference and Confidence Scores

At inference time:
1. Test audio goes through the same preprocessing and feature extraction pipeline
2. Features are normalized using the saved `scaler_1039.pkl`
3. The model outputs a raw logit which is passed through sigmoid to get `P(native | audio)`
4. Predicted label: `1` if probability > 0.5, else `0`
5. Confidence score: the raw sigmoid probability

Output format in `submissions/predictions.csv`:
```
file_id, predicted_label, confidence
288, 1, 0.923
294, 0, 0.711
...
```

## 9. Robustness Considerations

**Label noise** — Near-native speakers are genuinely ambiguous. Labels for bilingual or heritage speakers may be inconsistent. We addressed this via `pos_weight` class balancing and dropout regularization.

**Recording variability** — External datasets have different microphones, rooms, and recording conditions than the primary dataset. Explicit noise features give the model visibility into recording quality so it can distinguish acoustic artifacts from accent patterns. Audio normalization at preprocessing also reduces variability.

**Overfitting** — With limited training data, overfitting is the primary risk. We use dropout (0.3), early stopping via model checkpointing on val accuracy, and learning rate reduction on plateau.

**Generalization across languages** — The problem states recordings are in the same language but evaluation may include conditions not seen in training. WavLM-Large's pre-training on diverse speech data provides strong generalization, and prosody features are language-agnostic (rhythm and pitch patterns are universal accent cues).

## 11. What We Would Do With More Time

- **Fine-tune WavLM-Large** — unfreeze the last 3 transformer layers and jointly train with the classifier. This would allow the embeddings to specialize for accent classification rather than generic speech representation.
- **Temperature scaling** — post-hoc calibration of confidence scores to make probabilities more reliable
- **SpecAugment** — frequency and time masking during training for better robustness
- **Ensemble** — combine predictions from multiple trained models (mean pooling, GRU-only, full model) for more robust final predictions
- **Hierarchical attention** — separate attention over phoneme-level and phrase-level patterns
- **More external data** — larger and more diverse external datasets would significantly help generalization
